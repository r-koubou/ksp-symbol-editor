contexts - コンテキスト管理
========================


## このフォルダの概要

このフォルダには、セッションのコンテキストに関するドキュメントが格納されている。

- 今現在のセッションのコンテキストは progress.md に記載されている。
- 過去分のセッションのコンテキストは archives フォルダにYYYY-MM-DD.md 形式で保存されているが、AI側からの参照は不要である。

## progress.md について

- 「累積ログ」ではなく「現在のスナップショット」として使う
- **更新はAI側が作業完了後に行う。**

書式:

```markdown
# 現在の状態（2026-02-17 12:34 更新）

## 現状
- xxx は完了。src/editors/xxx.ts に実装済み

## 未完了
- yyy のコールバック処理

## 重要な判断・制約（なぜそうなっているか）
- ○○はあえて△△方式にしている。理由：□□

## 次回やること
1. yyy の続き
```

## アーカイブに関する事項

### アーカイブ条件

progress.md を更新する前に以下を確認し、
いずれかに該当する場合は先にアーカイブを作成する。

1. progress.md が 100 行を超えている
2. 人間から明示的にアーカイブを指示された

### アーカイブファイル名

archives/ に保存するファイル名は、progress.md のヘッダーに記載されている「最終更新日時（JST）」を使用する。

例: `「# 現在の状態（2026-02-18 14:30 更新）」` → `2026-02-18.md`
