tasks - タスク管理
================


## このフォルダの概要

このフォルダには、タスク管理に関するドキュメントが格納されている。

- 今現在のタスク内容は current.md に記載されている。
- 過去分のタスク内容は logs フォルダにYYYY-MM-DD.md 形式で保存されているが、AI側からの参照は不要である。

## current.md について

- **タスク作成は人間がまず行う**
- **AI側は作業中に完了したタスクの項目についてチェックボックスをオンにするだけでよい。**

書式:

```markdown
# ＜ゴール概要＞

## ゴールの詳細

＜概要以上に詳細な指定がある場合に記載する＞

## タスク内容

- [ ] ＜タスクA＞
- [ ] ＜タスクB＞
- [ ] ＜タスクC＞
:
:

## スコープ外（触らないこと）

- ＜触らないファイルやモジュール＞
```

## スコープ外（触らないこと）について

例:

```
- src/core/ 以下のファイル
- docs/specs/ のファイル（読むのはOK、変更はしない）
- variable エディタ以外のエディタ実装
```

など
- リファクタリングや大きな機能追加の時
- 複数エディタが並行して存在する時
- 「今回はこれだけやる」という境界を明確にしたい時

逆に小さな作業（1ファイルのみ）の場合は省略してもOK。
